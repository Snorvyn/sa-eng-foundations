# Contributing

## Branching
- Never commit to `main` directly.
- Use `feat/<short>`, `fix/<short>`, or `chore/<short>` branch names.

## Commit messages
Use Conventional Commits:
- `feat:` new capability
- `fix:` bug fix
- `chore:` tooling or housekeeping
- `docs:` docs-only
- `test:` tests-only
- `refactor:` internal change without behavior change

Example: `feat(math): add subtract() and tests`

## PRs
- Link any issues/tickets if applicable.
- Include what changed and why.
- Include test notes (how verified).
- Expect at least one review before merge.
